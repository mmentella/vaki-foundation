# Validation Checklist

## Status

This checklist is a practical aid for recording validation work.

It is non-canonical with respect to cognitive semantics, non-normative, adaptable by each Manifestation, not a mandatory process, and not a new semantic owner.

The canonical role of validation is defined in the [Foundation Validation Strategy](VALIDATION_STRATEGY.md). Evidence semantics remain owned by the [Evidence Model](../thinking/EVIDENCE.md). Foundation/Manifestation boundary semantics remain owned by the [Foundation Specification](../FOUNDATION_SPEC.md).

Use, shorten, adapt, or replace this checklist when local context warrants. The required property is semantic clarity, not completion of a form.

## Outcome

Use bounded verdict language consistent with the validation record's scope.

Possible verdict wording:

- Validation success within declared scope.
- Validation success with non-blocking observations.
- Inconclusive within declared scope.
- Local Manifestation issue observed.
- Foundation-level inquiry candidate.
- Foundation review required.

These are optional phrases, not universal mandatory verdicts. A validation record may use free-form wording when it better preserves the evidence and limits.

Identifying a Foundation-level inquiry candidate is not promotion, approval, or Foundation change.
